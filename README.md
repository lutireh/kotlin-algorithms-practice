# Algorithms & Problem Solving Practice

This repository contains my personal solutions to algorithm and data structure problems solved while preparing for technical interviews.

The exercises come from a variety of sources and focus on strengthening problem-solving skills commonly required in software engineering challenges.

Rather than organizing problems by platform, the goal is to build a consistent record of solved problems and document different approaches and patterns over time.

---

## Goals

This repository is used to:

- Improve algorithmic thinking
- Practice problem-solving patterns
- Prepare for technical interviews
- Keep a documented history of solved problems
- Write clean and well-structured Kotlin implementations

---

## Language

All solutions in this repository are implemented in:

- **Kotlin**

---

## Current Structure

For now, all problems are stored in a single directory while I focus on solving and documenting solutions.

---

## Solution Format

Each solution follows a simple and consistent structure inside the source code.

Example:

```kotlin
/**
 * Problem:
 * Given an array of integers nums and an integer target,
 * return the indices of the two numbers such that they add up to target.
 *
 * Approach:
 * Use a hash map to store previously seen values and check whether
 * the complement (target - current value) already exists.
 *
 * Time Complexity: O(n)
 * Space Complexity: O(n)
*/
```
